Codeup 3480 외식의 빗자루

by qkrwhdah03

2020. 05. 03
in Solving

문제 출처

< 문제 >

여느 날처럼 코드업을 즐기고 있던 sean9892는 외식을 하기로 결정했다. sean9892는 외식을 나갈 때에는 외식의 빗자루를 사용하는데, 외식의 빗자루는 시동을 걸 때마다 몇 가지 절차를 거쳐야 한다.

  • 시동을 건 후 출발지와 도착지를 입력한다.
  • 오늘이 a월 b일일때, 출발지와 도착지까지 정확히 a+b번 도로를 거쳐가는 경로의 수를 입력한다.
    1,2 번의 절차를 걸치면 외식의 빗자루는 목적지까지 아주 빠르게 도착할 수 있다. 다행히도 sean9892의 손에는 지도가 쥐어져 있고, 도로는 오직 두 건물의 사이만을 연결하고 있으며 분기점이 없으며 서로 만나지 않는 직선 도로이다. sean9892가 즐거운 외식을 만끽할수 있도록 도와주자.
입력

첫 줄에는 날짜를 의미하는 두 정수 a와 b가 공백 한 칸을 사이에 두고 입력된다. (단, a월 b일은 실제로 존재하는 날짜로만 입력된다.) 그 다음 줄에는 건물의 수 n과 도로의 수 m이 공백 한 칸을 사이에 두고 입력된다.(1<=n<=250, 0<=m<=40000) 3번째 줄부터 m개의 줄에 걸쳐 두 건물의 번호 xi와 yi가 공백 한 칸을 사이에 두고 입력된다. (1<=xi,yi<=n) 이는 건물 xi와 건물 yi가 도로로 직접 연결되어있음을 의미한다. 동일한 간선이 여러 번 입력될 수 있으며, 이는 하나의 간선으로 처리한다. 마지막 줄에는 출발지 s와 도착지 e가 공백 한 칸을 사이에 두고 입력된다. 출발지와 도착지는 같을 수 있다.

출력

첫 줄에는 출발지와 도착지의 번호를 출력한다. 두번째 줄에는 s에서 e로 가는 경로의 수를 출력한다. 경로가 많을 수 있으니 경로의 수를 30011로 나눈 나머지를 출력한다.

< 풀이 >

다이나믹 프로그래밍을 이용하면 간단하게 풀 수 있다. DP[k][n]을 k번 노드까지 n개의 건물을 거쳐 도착하는 방법의 수라고 정의하면, DP[k][n]은 k와 연결된 모든 노드 i들에 대해 DP[i][k-1]를 더해주는 형태로 계산이 가능하다. 단, 초기값의 경우 시작 건물 s의 DP[s][0]=1, 나머지는 0으로 설정해주면 된다. 코드를 보면 다음과 같다.

< 코드 >

  #include <iostream>
  #include <vector>
  #define N 30011
  using namespace std;

  int a,b,n,m,s,e;
  int DP[251][45]={0,}, check[251][251]={0,};
  vector<vector<int> > map;

  int f(int st, int end, int move){
  DP[st][0]=1;
  for(int i=1; i<=move; ++i){
    for(int j=1; j<=n; ++j){
      for(int k=0; k<map[j].size(); ++k){
        DP[j][i]+=DP[map[j][k]][i-1];
        DP[j][i]%=N;
      }
    }
  }
  return DP[end][move];
  }

  int main(){
    ios_base::sync_with_stdio(false); cin.tie(NULL);cout.tie(NULL);
    cin>>a>>b>>n>>m;
    map.resize(n+2);
    for(int i=0; i<m; ++i){
      int x,y;
      cin>>x>>y;
      if(!check[x][y]){
        map[x].push_back(y);
        if(x!=y) map[y].push_back(x);
      }
      check[x][y]=check[y][x]=1;
    }
    cin>>s>>e;
    cout<<s<<' '<<e<<'\n';
    cout<<f(s,e,a+b)%N;
  }
