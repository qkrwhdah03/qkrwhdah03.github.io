Centrarium Theme에 대해

by qkrwhdah03

2020. 05. 03
in ETC

서론

이 Github 블로그에 Centrarium Theme을 적용하기 전에는 내가 직접 Theme을 제작하고 있었다. 블로그를 만드는 김에 HTML, CSS와 같은 기본적인 프론트엔드 문법들을 익히고, 나만의 Theme을 만들어 보려고하였으나, 위의 두 가지 언어뿐 아니라 java-script, liquid 등도 배우고 적용해야 되었다. 그래서, 열심히 공부해서 전체적인 레이아웃도 구성하고, 사이드바나, 댓글, 반응형 웹 등의 기능을 만들었지만 혼자서 다 만들기엔 너무 시간이 낭비이고, 비효율적(웹의 구현도…)이라고 생각되서 결국 무료 Theme을 이용하게 되었다.

무료 Theme은 다음 사이트에서 찾아볼 수 있다.
To jekyll theme
내가 사용한 Centrarium Theme은 여기로 들어가면 다운받아 사용할 수 있다.

이러한 Theme의 기본적인 사용방법의 경우 다른 블로그나 공식 사이트에 잘 나타나있으므로, 이 글에서는 다루지 않는다. 더 알아보고 싶다면 여기로 가길 바란다.

이 글에서는 내가 Centrarium Theme을 사용하면서 만난 문제점들에 대해 간단히 정리하려고 한다. 혹시라도 누군가 나와 같은 문제를 만났을 때, 이 글을 보고 해결하길 바라면서…
참고로 이 글은 2020년 5월 3일 기준으로 작성되었다는 것을 알아두길 바란다.

문제점

* Pagination 오류

config.yml 파일의 상단부를 보면 다음과 같이 나온다.

plugins:
  - jekyll-archives # Sorry, not GitHub pages friendly!
  - jekyll-sitemap
  - jekyll-paginate-v2

현재, Centrarium은 jekyll-paginate-v2 의 plugin을 사용하고 있다. 이 plugin은 jekyll serve 를 통해서 호스팅을 했을 때는, 정상적으로 작동하였지만 Github에 Push해 주었을 때, 포스트가 뜨지않게되는 오류가 있었다. 이를 해결하기 위해서는 jekyll-paginate-v2 를 jekyll-paginate 으로 바꾸어주어 해결하면 된다. 또, 밑에 Pagination 설정의 경우

#pagination:
#  enabled: true
#  per_page: 5
#  permalink: "/page/:num/"
#  sort_reverse: true

paginate: 5
paginate_path: "/page/:num"

으로 바꾸어주면 된다.

* 코드 하이라이트

정말 이상하게도 일부 코드에 대해 코드 하이라이트가 잘 적용이 되지 않았다. Centrarium은 CDN방식으로 코드 하이라이트 css파일을 link해주고 있는데, 이 것이 잘 적용되지 않았던 것 같다. 이를 해결하기 위해서는 include 폴더의 head.html 파일로 가서

 <link rel="stylesheet" href="//cdnjs.cloudflare.com/ajax/libs/highlight.js/9.9.0/styles/monokai.min.css">

의 부분을 js파일로 변경시켜주었더니 해결되었다. 이 링크에서 원하는 Theme을 골라 다음과 같이 변경시켜준다.

<link rel="stylesheet" href="//cdnjs.cloudflare.com/ajax/libs/highlight.js/9.2.0/styles/monokai_sublime.min.css"> <!--여기서 theme의 이름만-->
<script src="//cdnjs.cloudflare.com/ajax/libs/highlight.js/9.2.0/highlight.min.js"></script>
<script>hljs.initHighlightingOnLoad();</script>

위에는 monokai_sublime 을 적용시켜준 코드 이다. 앞서말했듯이, 위에서 원하는 Theme을 골라서 이름만 바꾸어주면 된다.

* 대문자 문제

카테고리, 태그 등을 포스팅에 이용하려고 영어로 작성하는 경우에 단어의 맨 앞 단어만 대문자가 되고 나머지는 무시되어 표현되었다. 이는 layout, index.html 등의 파일에서 일부 liquid 문법을 삭제해주면 해결된다. 다음과 같다.


  {{ cat | capitalize }}
  <!--파일들을 살펴보면 이렇게 쓰여있는 곳이 있다. 이를 아래와 같이 바꾼다-->
  {{ cat }}


* 날짜 표시

포스팅에서 date변수 설정시 5월이 넘어가게 되면 포스팅이 올라가지 않는 오류가 있었다. 이것의 원인을 찾지는 못했고, post의 상단에 변수를 새로하나 만들어주어 그 날자를 출력하도록 바꾸어 주었다.

---
layout: post
title: "Centrarium Theme에 대해"
date: 2020-01-01 00:00:00
pdate: 2020-05-03 8:56:00
author: qkrwhdah03
categories: ETC
tags: blog centrarium jekyll github
---

위는 이 포스팅의 상단부분을 복사한 내용이다. 위와 같이 pdate를 설정해주어 출력하도록 바꾸었다.

* Archieve 오류

포스트를 보면 Tag를 통해 같은 Tag의 글들을 볼 수 있도록 되어있다. 하지만, 이 오류도 아까의 Pagination 오류와 마찬가지로 jekyll serve 로 호스팅하였을 때는 정상적으로 작동하였지만, Github에 Push하여 들어가보면 Tag를 클릭하였을 때, 404오류 페이지를 띄운다. 이 원인은 Plugin을 보면 jekyll-archives 가 있는 것을 볼 수 있다.

plugins:
  - jekyll-archives # Sorry, not GitHub pages friendly!
  - jekyll-sitemap
  - jekyll-paginate-v2

우리가 알아야하는 것은 jekyll-archives 는 Github에서 지원해주지 않는다. 이를 해결하기 위해서는 로컬에서 jekyll serve 를 통해 만들어진 site 폴더의 내부 내용을 Github의 Repo로 직접 Push해주면 된다.

글을 마치며

이 문제들을 해결하는데 정말 하루종일 찾아보았던 것 같다. 이러한 문제점들이 있었긴 했지만, 정말 디자인이 깔끔하고 파일들의 구조또한 이해하기 쉬운형태로 되어있는 좋은 Theme인 것 같다. 혹시라고 같은 Theme을 사용하게 된다면, 이 글을 통해 나와 같은 시간낭비를 하지 않길 바란다.
