### 1. Markdown이란?
Markdown은 텍스트 기반의 마크업언어로 2004년 존 그루버에 의해 만들어졌다. 특수기호와 문자를 이용한 간단한 구조의 문법으로 웹에서 보다 빠르게 컨텐츠를 작성하고 보다 직관적이라는 장점이 있다.  
<br>

### 2.  Markdown 문법 알아보기

* ####  제목
제목을 표시할 때는 "="나 "-"를 통해 지정한다. 다음과 같이 사용한다.

```

  h1 : 가장 큰 제목
  ===
  h2 : 다음 큰 제목
  ---

```

* #### 글머리
"#"의 개수를 통해 글머리를 지정할 수 있다. "#"의 개수는 1~6개로 개수가 늘어날 때마다 글씨의 크기가 작아지고 하위 분류로 내려간다.

``` c

  #h1
  ##h2
  ###h3
  ####h4
  #####h5
  ######h6

```
* #### 인용구
인용구의 경우 ">"의 문자를 사용하여 표현한다. ">"문자 뒤에 인용구를 붙이면 된다.

* #### 목록(리스트)
"\*" 모양을 사용하여 순서 없는 리스트를 만들고, "1."과 같이 숫자 뒤에 점을 붙여 순서 있는 리스트를 만들어낸다.

* #### 코드
숫자 1키 옆에 있는 "\`"기호나 "\-" 기호를 사용한다.

``` c

  --- pyhton
  import numpy as np
  x=np.arange(-5,5,0.1)
  y=np.sin(x)
  ---

```
이런 식으로 연속되게 기호 3개를 입력하고 사용된 언어의 이름을 옆에 붙여준 후, 사이에 코드를 입력해 주면 된다.

* #### 수평선
"-" 이나 "\*" 이나 "\_" 를 3개 이상 작성하면 수평선이 된다.  
(단, -을 사용할 경우에 header로 인식할 수 있으니 이 전 라인은 비워두어야 한다.)

* #### 링크

``` c

  [링크](주소 "링크에 대한 설명-마우스 hover시에 나타남")

```
위의 형태로 작성하여  \<a\> 태그와 같은 역할을 한다.

* #### 강조

``` c

  - 기울어진 글씨체 -
  -- 굵은 글씨체 --

  * 기울어진 글씨체 *
  ** 굵은 글씨체 **

```


* #### 이미지
인라인 이미지: ![alt text] (/test.png)  
링크 이미지 : ![alt text] (image_URL)  
참조 이미지 : ![alt text] [1] , [1]:/test.png

* ### Markdown 포스트
내가 사용하는 Github 블로그의 경우 테마에 따라 다르겠지만, file.md와 같이 markdown 파일의 시작부분에 title 과 같이 post에 대한 정보를 미리 적어주어야한다. 그후에 위의 문법들을 사용하여 컨텐츠를 표시해 주면 된다. 마지막에는 .md파일을 \_posts 폴더에 넣어서 Github에 변경사항을 push 해주면 된다.
